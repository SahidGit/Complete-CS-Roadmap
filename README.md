## Why this repo

We focus on practical, up-to-date learning:

- Learn by building: each week ends with a small project
- Official resources: links point to official docs (MDN, RFCs, vendor docs)
- Community maintained: open to improvements and PRs

## How to get started

1. Fork or star the repo on GitHub
2. Clone locally:

```bash
git clone https://github.com/SahidGit/Complete-CS-Roadmap.git
cd Complete-CS-Roadmap
```

3. Start with the overview: [ROADMAP.md](ROADMAP.md) and then open `[weeks/Week-01.md](weeks/Week-01.md)`

---

## Recommended workflow

- Follow the weekly order — each week builds on the previous one
- Do the mini-projects and push them to your own GitHub portfolio
- Use `PROGRESS-TRACKER.md` to mark what you've completed

---

## Contribution guide (short)

We love contributions. Quick checklist:

1. Read [CONTRIBUTING.md](CONTRIBUTING.md)
2. Open an issue before large changes
3. Create a branch: `git checkout -b feat/some-improvement`
4. Keep changes focused and test links/resources
5. Submit a PR and link the issue

If you're adding large binary assets (images, fonts, videos), please use Git LFS or host them externally and link to them.
