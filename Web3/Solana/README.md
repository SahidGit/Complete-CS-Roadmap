# ☀️ Solana Programming Reference

Solana is a high-performance, single-layer blockchain network designed for scalable decentralized applications.

---

## 🏗️ Core Architecture (SVM)

### 1. Account Model
Everything on Solana is stored inside **Accounts**:
- Accounts hold a metadata header (including lamports/SOL balance, owner program, executable flag) and a raw byte data array.
- **Executable Accounts**: Hold compiled SVM bytecode (Programs).
- **Data Accounts**: Owned by a program. Hold variables (state) populated by users.

### 2. Program Derived Addresses (PDAs)
PDAs are specialized keys generated by hash operations off the Ed25519 elliptic curve:
- They do not possess matching private keys.
- Used by programs to programmatically sign transactions and enforce strict database record associations (e.g. mapping a user to their vault using seeds).

```
Seeds (e.g., "user-vault", User PublicKey) + Program ID ──► [ hash ] ──► PDA Address (Off Curve)
```

### 3. Client Bindings (@solana/web3.js)
Interacting with the RPC cluster from a Javascript/TypeScript frontend:
```javascript
const { Connection, PublicKey, clusterApiUrl } = require('@solana/web3.js');

// Create connection to Devnet
const connection = new Connection(clusterApiUrl('devnet'), 'confirmed');

// Query lamports balance of an account
async function checkBalance() {
  const pubkey = new PublicKey('3b7d...');
  const balance = await connection.getBalance(pubkey);
  console.log(`Balance: ${balance / 10**9} SOL`);
}
```
