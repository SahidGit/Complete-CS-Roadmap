# 🤝 Contributing

Thank you for improving this roadmap. Contributions must be accurate, beginner-friendly and maintainable.

1. Check existing issues and discussions.
2. Prefer official, stable sources; explain non-official sources.
3. Keep one focused purpose per pull request.
4. Check every Markdown link and preview the document.
5. Update `CHANGELOG.md` under **Unreleased**.

Use concrete learning outcomes, inclusive language and verified resources. Do not add affiliate links, copied course notes, or unverified claims.
