# Week 03 — CSS and Responsive Design

## Goal
Turn semantic pages into robust layouts that work across screens and user preferences.

## Learning Objectives
- Use the cascade, box model, Flexbox and Grid deliberately.
- Build mobile-first responsive layouts and purposeful animation.
- Understand Tailwind utility concepts without depending on them blindly.

## Topics and Concepts
Selectors, inheritance, specificity, custom properties, sizing, typography, Flexbox, Grid, media queries, container-aware thinking, transitions, keyframes and reduced motion.

## Resources
- **Official documentation:** [MDN CSS](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Styling_basics), [Tailwind docs](https://tailwindcss.com/docs).
- **Course/video:** [Kevin Powell](https://www.youtube.com/@KevinPowell).
- **Practice/book:** [Frontend Mentor](https://www.frontendmentor.io/), *Every Layout* by Heydon Pickering and Andy Bell.

## Interview Questions
1. How do Grid and Flexbox solve different layout problems?
2. What causes a specificity conflict?
3. How do you respect `prefers-reduced-motion`?

## Mini Projects
1. **Netflix Clone UI** — intermediate, 8h. Responsive media browsing interface. *Skills:* Grid, visual hierarchy. *Extension:* add skeleton states.
2. **Responsive Portfolio** — intermediate, 7h. Portfolio that adapts from phone to desktop. *Skills:* fluid type, navigation. *Extension:* add dark mode.
3. **Restaurant Website** — intermediate, 8h. Menu and reservation layout. *Skills:* forms, responsive imagery. *Extension:* add accessible modal behavior.
