# Week 02 — Semantic HTML

## Goal
Create accessible, meaningful document structure before styling it.

## Learning Objectives
- Use semantic HTML, forms, tables and media appropriately.
- Apply accessibility basics and introductory SEO metadata.
- Validate HTML and test keyboard navigation.

## Topics and Concepts
Document outline, landmarks, headings, links, images and alt text, forms and labels, native validation, tables, audio/video, metadata and Open Graph.

## Resources
- **Official documentation:** [MDN HTML](https://developer.mozilla.org/en-US/docs/Learn_web_development/Core/Structuring_content), [W3C WAI](https://www.w3.org/WAI/), [Google Search Central](https://developers.google.com/search/docs).
- **Course/video:** [freeCodeCamp Responsive Web Design](https://www.freecodecamp.org/learn/2022/responsive-web-design/).
- **Practice/book:** [Frontend Mentor](https://www.frontendmentor.io/), *HTML and CSS* by Jon Duckett.

## Interview Questions
1. When should a button be used instead of a link?
2. What makes an image alternative text useful?
3. Why is native HTML usually preferable to ARIA?
