# Week 01 — Engineering Tools and the Internet

## Goal
Work confidently in a terminal, collaborate with Git, and explain how a browser reaches a web server.

## Learning Objectives
- Use Git branches, commits, remotes and pull requests.
- Navigate Linux-like shells and manage files safely.
- Explain DNS, HTTP, HTTPS, SSH and basic networking.

## Topics and Concepts
Git/GitHub, Linux CLI, files and permissions, networking, IP and DNS, HTTP methods/statuses, TLS, SSH keys, Markdown, VS Code and focus habits.

## Resources
- **Official documentation:** [Git Book](https://git-scm.com/book/en/v2), [GitHub Docs](https://docs.github.com/), [MDN HTTP](https://developer.mozilla.org/en-US/docs/Web/HTTP), [OpenSSH](https://www.openssh.com/manual.html).
- **Course/video:** [GitHub Skills](https://skills.github.com/) and [freeCodeCamp](https://www.youtube.com/@freecodecamp).
- **Practice/books:** [Learn Git Branching](https://learngitbranching.js.org/), *Pro Git*.

## Interview Questions
1. What is the difference between `git merge` and `git rebase`?
2. Why does HTTPS need certificates?
3. What happens after entering a URL in a browser?

## Mini Projects
1. **Linux Cheat Sheet** — beginner, 4h. Document navigation, permissions, pipes and safe file handling. *Skills:* CLI, Markdown. *Extension:* include a troubleshooting section.
2. **Git Portfolio** — beginner, 5h. Create a repository demonstrating branches, PRs and releases. *Skills:* GitHub workflow. *Extension:* add GitHub Actions Markdown checks.
3. **Terminal Commands Guide** — beginner, 4h. Write task-based command recipes. *Skills:* command-line research. *Extension:* include Windows PowerShell equivalents.
